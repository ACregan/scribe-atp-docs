**Scribe** is a writing and publishing tool built for the open web. You write and organise your content in Scribe CMS, and it becomes available to any website you or a developer sets up to display it.

Your content is stored in your own account — you own it. If you stop using Scribe tomorrow, your writing doesn't disappear with it.

## What you can do

<CardGrid>
  <Card title="Write articles" icon="pencil">
    Compose long-form content with a rich text editor. Add images, format headings, lists, quotes, and code blocks.
  </Card>
  <Card title="Organise into sites and groups" icon="open-book">
    Create one or more sites — one per domain — then organise your articles into groups (sections or categories) within each site.
  </Card>
  <Card title="Control what's published" icon="approve-check">
    Articles move through three states: Draft → Unpublished → Published. You control when something goes live.
  </Card>
  <Card title="Power multiple websites" icon="rocket">
    One Scribe account can serve multiple websites simultaneously. A developer installs the Scribe SDK on each site and it reads from your account.
  </Card>
</CardGrid>

## How to use these docs

If you're new to Scribe, start with **[The Content Model](/authors/content-model)** to understand how Sites, Groups, and Articles fit together. Then follow **[Getting Started](/authors/getting-started)** to set up your first site and publish your first article.

If you already have a site set up and want to do more with it, go straight to **[Managing Your Site](/authors/managing-your-site)**.

If you want to display your content on a website but aren't a developer yourself, see **[Using a Website](/authors/using-a-website)** — it explains what to ask a developer for and what they'll need from you.
